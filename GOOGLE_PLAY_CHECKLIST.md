# ✅ Google Play Store Checklist

## 📋 Trước khi Submit

### 1. Google Play Developer Account
- [ ] Đăng ký tài khoản: https://play.google.com/console/signup
- [ ] Thanh toán $25 (một lần, trọn đời)
- [ ] Xác thực email
- [ ] Điền thông tin developer

### 2. App Requirements

#### Technical
- [ ] Target SDK: Android 13 (API 33) trở lên
- [ ] Min SDK: Android 5.0 (API 21) trở lên
- [ ] App Bundle (.aab) format
- [ ] 64-bit support (arm64-v8a, x86_64)
- [ ] ProGuard/R8 enabled (code obfuscation)
- [ ] Signing key (keystore file)

#### Content
- [ ] App không vi phạm chính sách Google
- [ ] Không có nội dung bạo lực, khiêu dâm
- [ ] Không có malware, virus
- [ ] Không lừa đảo, spam

### 3. Store Listing Assets

#### Required
- [ ] **App Icon** (512x512px, PNG, 32-bit)
- [ ] **Feature Graphic** (1024x500px, PNG/JPEG)
- [ ] **Screenshots** (2-8 ảnh, 1080x1920px)
- [ ] **App Name** (tối đa 30 ký tự)
- [ ] **Short Description** (tối đa 80 ký tự)
- [ ] **Full Description** (tối đa 4000 ký tự)

#### Optional
- [ ] Promo Video (YouTube link)
- [ ] Tablet Screenshots
- [ ] TV Banner
- [ ] Wear OS Screenshots

### 4. Legal Documents

#### Required
- [ ] **Privacy Policy** (URL công khai)
  - Dữ liệu thu thập
  - Cách sử dụng
  - Cách bảo vệ
  - Quyền người dùng
  
- [ ] **Data Safety Form**
  - Loại dữ liệu thu thập
  - Mục đích sử dụng
  - Có chia sẻ với bên thứ 3 không
  - Có mã hóa không

#### Optional
- [ ] Terms of Service
- [ ] EULA (End User License Agreement)

### 5. App Information

- [ ] **Category**: Productivity
- [ ] **Tags**: time management, productivity, task manager, pomodoro
- [ ] **Content Rating**: Everyone / Teen
- [ ] **Target Age**: 13+
- [ ] **Contact Email**: support@timemanager.com
- [ ] **Website**: https://timemanager.com
- [ ] **Phone Number** (optional)

### 6. Pricing & Distribution

- [ ] **Free** hoặc **Paid**
- [ ] **Countries**: Chọn quốc gia phân phối
- [ ] **In-app purchases**: Có/Không
- [ ] **Ads**: Có/Không

### 7. Testing

#### Internal Testing (Alpha)
- [ ] Upload APK/AAB
- [ ] Thêm testers (email)
- [ ] Test 1-2 tuần
- [ ] Fix bugs

#### Closed Testing (Beta)
- [ ] Mở rộng testers
- [ ] Test 2-4 tuần
- [ ] Collect feedback
- [ ] Polish UI/UX

#### Open Testing (Optional)
- [ ] Public beta
- [ ] Unlimited testers
- [ ] Final testing

### 8. Pre-launch Report

Google tự động test app trên:
- [ ] Nhiều devices
- [ ] Nhiều Android versions
- [ ] Crash detection
- [ ] Security vulnerabilities
- [ ] Performance issues

Review report và fix issues!

### 9. Release Management

- [ ] **Version Code**: 1, 2, 3... (tăng dần)
- [ ] **Version Name**: 1.0.0, 1.0.1... (semantic versioning)
- [ ] **Release Notes**: Mô tả tính năng mới
- [ ] **Rollout**: 
  - Staged rollout (5% → 10% → 50% → 100%)
  - Hoặc full rollout (100%)

### 10. Post-launch

- [ ] Monitor crash reports
- [ ] Respond to reviews
- [ ] Track analytics
- [ ] Plan updates

## 💡 Tips

1. **Start with Internal Testing** - Don't rush to production
2. **Use Staged Rollout** - Release to 5% first, then increase
3. **Monitor Crash Reports** - Fix crashes immediately
4. **Respond to Reviews** - Show you care about users
5. **Update Regularly** - Monthly updates keep users engaged
6. **Localize** - Add Vietnamese language support
7. **ASO (App Store Optimization)** - Use keywords in title/description
8. **Screenshots Matter** - Make them beautiful and informative

---

## 🚫 Common Rejection Reasons

1. Missing privacy policy
2. Incomplete data safety form
3. Crashes on startup
4. Broken features
5. Misleading screenshots
6. Copyright violations
7. Low quality assets
8. Spam/malware detected

**Avoid these to get approved faster!**
